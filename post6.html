About evaluating accessibility

Excited to share my thoughts and findings

Posted by Cuong Mai on May 10, 2020

The site/domain that I am evaluating for this report is Google

The purpose of this site is for all users to use for shopping, news, entertainment, email, and many more. the intended audiences are everyone and every age.

The tool I am using is from this website, the site listed 10 examples of compliance accessibility for any websites/developers to use a guideline. DBS Interactive

Here are the ten accessibility examples I am using to evaluate the site.

1. Contrast ratio

2. Don’t rely on color

3. Label forms clearly

4. Provide feedback on errors and omissions

5. No mouse, no problem

6. Keep navigation consistent

7. Simple heading and spacing

8. Design for different devices, views, and screens

9. Offer alternatives for consuming media

10. Give users control

1. Contrast ratio -The fonts of the site are easy to read, the background color does not interfere with the fonts.

2. Don’t rely on color -The site background color is white, the background color of the fonts is gray, and overall there is not too much color on the site that can hinder one ability to browse and read especially for colorblind users.

3. Label forms clearly -The search bar is in the middle of the homepage with a magnify glass symbol to designate for searching. Underneath the search bar, there are two options for user to click after a search phrase is entered.

4. Provide feedback on errors and omissions -If a user does not enter a search keyword and click on the search icon, there is no error message or any message to tell the user to

5. No mouse, no problem -Users can use the tab key to navigate around the site without using a mouse or a keyboard mouse pad as well as the arrows to move up and down of the pages.

6. Keep navigation consistent -The site options, heading, and setting are at the corners of the page. When navigating to other pages such as news, mail, etc. these options remain consistent throughout to help users quickly use to navigate.

7. Simple heading and spacing -The homepage have plenty of spacing between the page menus, options, and the search bar. The news homepage headlines are in black fonts and white background, this simple layout and design allow all users to read and see the headlines.

8. Design for different devices, views, and screens -Users can use a tablet or mobile devices to access the site without any issues or use the mobile apps.

9. Offer alternatives for consuming media -The Youtube site, there are subtitle and closed captions options available to users if one is having issue with audio.

10. Give users control -I think the site give users a lot of control to navigate and use the site whether it is the news, videos, or email, the users have full control.

Conclusion

I think the only problem I found on the site is example number four, provide feedback on errors and omissions, if user don't input a search keyword and clicking on the search option does not do anything, perhaps the site can have a message to remind user to make sure to input a search keyword before clicking on the search icon. At this I still don't know too much about web accessibility beside some of the simple one such as mobile access, fonts and color to see other problems the site might need to improve or implement. I think overall Google have done a good job on all the websites to allow all users access to the site and services. I am still trying to learn and understand the 508 rules and guidelines.
