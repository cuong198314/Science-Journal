Summary of using Wave and Lighthouse tool to ID accessiblity problems

Excited to share my thoughts and findings

Posted by Cuong Mai on May 17, 2020

The two tools that I am using to evaluate all my pages for accessibility issues are Wave and Lighthouse.

For some reasons I am really having trouble with this assignment to try and understand of the report and what I should do to correct them. Even from the reading I am just having difficulties to understand the full context of what exactly is a site accessibility compatible. Every single pages of the site have some type of problems and at this point I just don’t know how to go about fixing them to make it accessibility compatible. For example, the Wave report for the about page have errors from contrast to structural to ARIA. When I go to the details of trying to understand what are these errors and I just simply can’t grasp what am I supposed to fix to correct these errors. It would be great if there is an example for me to see the differences between the errors and the recommended changes to improve the site.

The two changes that I made to each page are adding text and link to the footer of the page for Twitter, Facebook, and Github logos to allow user access to each site. The other changes is making sure each image have accessible link to the site if it is applicable and text. I will need to spend more time on this topic to really get a better understanding of all of these errors from these tools and how to apply all the fixes to make the site better.
